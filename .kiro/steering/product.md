# Scrapling 产品概览

## 什么是 Scrapling？

Scrapling 是一个自适应、高性能的 Python 网页爬取框架，专为现代网络环境设计。它可以处理从单个 HTTP 请求到大规模分布式爬取的所有需求。

## 核心价值主张

- **自适应解析**：网站结构变化时自动重新定位元素
- **反机器人绕过**：内置支持 Cloudflare Turnstile 等反爬虫系统
- **多种获取策略**：HTTP 请求、浏览器自动化动态渲染和隐秘模式
- **生产级爬虫框架**：并发请求、多 Session 支持、暂停/恢复功能
- **AI 集成**：MCP 服务器用于 AI 辅助网页爬取和数据提取
- **性能优化**：92% 测试覆盖率和完整类型提示

## 主要特性

1. **获取器（Fetchers）**：多种网页内容获取策略

   - `Fetcher`：具有 TLS 指纹识别的快速 HTTP 请求
   - `StealthyFetcher`：具有浏览器自动化的高级反检测
   - `DynamicFetcher`：用于 JavaScript 密集型网站的完整浏览器自动化
2. **爬虫框架**：类似 Scrapy 的爬虫构建 API

   - 并发请求处理
   - 多 Session 支持（同一爬虫中的 HTTP + 浏览器）
   - 带检查点的暂停/恢复
   - 实时数据流模式
3. **解析**：丰富的元素选择和导航

   - CSS 选择器、XPath、BeautifulSoup 风格 API
   - 具有相似性匹配的自适应元素查找
   - 文本搜索和正则表达式支持
   - DOM 导航（父级、兄弟级、子级）
4. **开发者体验**

   - 交互式网页爬取 Shell
   - 无需代码的 CLI 直接提取
   - 完整的类型提示和 IDE 支持
   - 预构建镜像的 Docker 支持

## 项目开发约定

1、Agent回复必须输出简体中文

2、减少不必要的md文档，只需要每阶段任务完成后总结性md文档就可以，并且任务完成后在任务列表打勾表示完成

3、后端API产生的测试文档存放在backend/tests目录，项目文档放在backend/.prj_back;前端页面产生的测试文档存放在frontend/tests,项目文档存放在frontend/.prj_front

- 需要网页数据的 AI/ML 从业者
- 自动化工程师
- 收集网页数据的研究人员
- 构建数据管道的开发者
