# Scrapling Web UI 需求文档

## 介绍

Scrapling Web UI 是一个现代化的 Web 应用，为非技术用户提供图形化界面来使用 Scrapling 高性能网页爬取框架。该系统通过浏览器提供直观的爬虫任务管理、实时进度监控和可视化结果展示，使网页爬取变得易于访问。

## 术语表

- **Scrapling**：高性能 Python 网页爬取框架，支持多种获取策略
- **爬虫任务（Spider Task）**：用户定义的单个爬取操作，包含目标 URL、选择器和配置
- **获取器（Fetcher）**：Scrapling 中的网页内容获取策略（HTTP、动态、隐秘）
- **选择器（Selector）**：CSS 或 XPath 表达式，用于从 HTML 中提取特定元素
- **WebSocket**：双向通信协议，用于实时推送爬取进度和日志
- **会话（Session）**：用户的登录状态和相关配置
- **任务历史**：已完成或进行中的爬虫任务的记录
- **结果集（Result Set）**：单个爬虫任务的提取数据集合
- **FastAPI**：现代 Python Web 框架，用于构建后端 API
- **React**：前端 JavaScript 库，用于构建用户界面

## 需求

### 需求 1：用户认证与会话管理

**用户故事**：作为一个用户，我想通过用户名和密码登录系统，以便安全地访问我的爬虫任务和数据。

#### 验收标准

1. WHEN 用户在登录页面输入有效的用户名和密码，THE 系统 SHALL 验证凭证并创建会话
2. WHEN 用户输入无效的凭证，THE 系统 SHALL 显示错误消息并拒绝登录
3. WHEN 用户已登录，THE 系统 SHALL 在浏览器中存储会话令牌（JWT）
4. WHEN 用户点击登出按钮，THE 系统 SHALL 清除会话令牌并重定向到登录页面
5. WHEN 用户的会话过期，THE 系统 SHALL 自动重定向到登录页面
6. WHERE 用户选择"记住我"选项，THE 系统 SHALL 延长会话有效期至 30 天

### 需求 2：爬虫任务创建与配置

**用户故事**：作为一个用户，我想通过 Web 界面创建和配置爬虫任务，以便无需编写代码即可定义爬取操作。

#### 验收标准

1. WHEN 用户点击"新建任务"按钮，THE 系统 SHALL 打开任务创建表单
2. THE 任务创建表单 SHALL 包含以下字段：任务名称、目标 URL、获取器类型、CSS/XPath 选择器、请求超时、重试次数
3. WHEN 用户选择获取器类型，THE 系统 SHALL 显示该获取器的特定配置选项（例如，隐秘模式的反爬虫选项）
4. WHEN 用户输入 CSS 或 XPath 选择器，THE 系统 SHALL 提供实时验证反馈
5. WHEN 用户点击"保存任务"，THE 系统 SHALL 验证所有必填字段并保存任务配置
6. IF 任务配置无效，THEN THE 系统 SHALL 显示具体的验证错误消息
7. WHERE 用户选择"高级选项"，THE 系统 SHALL 显示代理配置、自定义请求头、Cookie 管理等选项

### 需求 3：爬虫任务执行与进度监控

**用户故事**：作为一个用户，我想实时查看爬虫任务的执行进度和日志，以便了解爬取操作的状态。

#### 验收标准

1. WHEN 用户点击"运行任务"按钮，THE 系统 SHALL 启动爬虫任务并显示进度页面
2. WHEN 任务运行中，THE 系统 SHALL 通过 WebSocket 实时推送进度更新（已处理 URL 数、成功率、当前 URL）
3. WHEN 任务运行中，THE 系统 SHALL 实时显示爬取日志（INFO、WARNING、ERROR 级别）
4. WHEN 用户点击"暂停"按钮，THE 系统 SHALL 暂停当前任务并保存检查点
5. WHEN 用户点击"恢复"按钮，THE 系统 SHALL 从检查点恢复任务执行
6. WHEN 用户点击"停止"按钮，THE 系统 SHALL 立即停止任务并保存已收集的数据
7. WHEN 任务完成，THE 系统 SHALL 显示完成状态、总耗时、成功/失败统计

### 需求 4：结果数据展示与导出

**用户故事**：作为一个用户，我想以多种格式查看和导出爬取结果，以便进行后续数据分析。

#### 验收标准

1. WHEN 任务完成，THE 系统 SHALL 在表格视图中显示提取的数据
2. THE 表格视图 SHALL 支持分页、排序和搜索功能
3. WHEN 用户点击表格中的行，THE 系统 SHALL 显示该行的详细信息
4. WHEN 用户点击"导出"按钮，THE 系统 SHALL 提供以下格式选项：CSV、JSON、Excel
5. WHEN 用户选择导出格式，THE 系统 SHALL 生成文件并触发浏览器下载
6. WHERE 用户选择"高级导出"，THE 系统 SHALL 允许选择特定列和应用过滤条件
7. WHEN 用户点击"复制"按钮，THE 系统 SHALL 将选定数据复制到剪贴板

### 需求 5：任务历史与管理

**用户故事**：作为一个用户，我想查看和管理我的历史爬虫任务，以便重复使用或修改之前的配置。

#### 验收标准

1. WHEN 用户访问"任务历史"页面，THE 系统 SHALL 显示所有历史任务的列表
2. THE 任务列表 SHALL 包含以下信息：任务名称、创建时间、最后运行时间、状态、结果数量
3. WHEN 用户点击任务，THE 系统 SHALL 显示任务的详细信息和配置
4. WHEN 用户点击"复制任务"，THE 系统 SHALL 创建该任务的副本并打开编辑表单
5. WHEN 用户点击"删除任务"，THE 系统 SHALL 显示确认对话框，删除后移除任务及其结果
6. WHEN 用户点击"重新运行"，THE 系统 SHALL 使用相同配置重新执行任务
7. THE 任务列表 SHALL 支持按状态、日期范围和关键词过滤

### 需求 6：获取器选择与配置

**用户故事**：作为一个用户，我想选择合适的获取器类型并配置其参数，以便适应不同网站的爬取需求。

#### 验收标准

1. WHEN 用户创建任务，THE 系统 SHALL 显示三种获取器选项：HTTP、动态、隐秘
2. WHEN 用户选择 HTTP 获取器，THE 系统 SHALL 显示基本配置（超时、重试、请求头）
3. WHEN 用户选择动态获取器，THE 系统 SHALL 显示浏览器配置（等待时间、JavaScript 执行、视口大小）
4. WHEN 用户选择隐秘获取器，THE 系统 SHALL 显示反爬虫配置（Cloudflare 解决、浏览器指纹、代理）
5. WHEN 用户悬停获取器选项，THE 系统 SHALL 显示该获取器的说明和适用场景
6. WHERE 用户选择隐秘获取器，THE 系统 SHALL 提供"解决 Cloudflare"复选框

### 需求 7：选择器验证与预览

**用户故事**：作为一个用户，我想在执行任务前验证选择器是否正确，以便避免无效的数据提取。

#### 验收标准

1. WHEN 用户在选择器字段中输入 CSS 或 XPath 表达式，THE 系统 SHALL 实时验证语法
2. IF 选择器语法无效，THEN THE 系统 SHALL 显示错误提示
3. WHEN 用户点击"测试选择器"按钮，THE 系统 SHALL 使用提供的 URL 和选择器进行测试爬取
4. WHEN 测试完成，THE 系统 SHALL 显示匹配的元素数量和预览数据
5. WHEN 用户点击"预览"，THE 系统 SHALL 显示网页的渲染视图并高亮匹配的元素

### 需求 8：代理与请求头管理

**用户故事**：作为一个用户，我想配置代理和自定义请求头，以便绕过地理限制和反爬虫检测。

#### 验收标准

1. WHEN 用户在高级选项中点击"代理配置"，THE 系统 SHALL 显示代理管理界面
2. THE 代理管理界面 SHALL 支持添加、编辑、删除代理
3. WHEN 用户添加代理，THE 系统 SHALL 验证代理地址格式（IP:端口）
4. WHEN 用户启用"代理轮换"，THE 系统 SHALL 在每个请求中轮换使用配置的代理
5. WHEN 用户点击"请求头管理"，THE 系统 SHALL 显示自定义请求头编辑器
6. THE 请求头编辑器 SHALL 支持添加、编辑、删除自定义请求头
7. WHEN 用户点击"导入 Cookie"，THE 系统 SHALL 提供从浏览器导入 Cookie 的选项

### 需求 9：仪表板与统计

**用户故事**：作为一个用户，我想在仪表板上查看爬虫活动的统计信息，以便了解系统的使用情况。

#### 验收标准

1. WHEN 用户访问仪表板，THE 系统 SHALL 显示以下统计信息：总任务数、本月任务数、成功率、总提取数据量
2. THE 仪表板 SHALL 显示最近 7 天的任务执行趋势图表
3. THE 仪表板 SHALL 显示最常用的获取器类型的饼图
4. WHEN 用户点击图表中的数据点，THE 系统 SHALL 显示该时期的详细任务列表
5. THE 仪表板 SHALL 显示系统资源使用情况（CPU、内存、网络）

### 需求 10：错误处理与恢复

**用户故事**：作为一个用户，我想在爬取过程中遇到错误时获得清晰的错误信息和恢复建议。

#### 验收标准

1. IF 爬虫任务失败，THEN THE 系统 SHALL 显示错误类型和详细错误消息
2. WHEN 用户遇到网络错误，THE 系统 SHALL 建议重试或更换代理
3. WHEN 用户遇到选择器错误，THE 系统 SHALL 建议修改选择器或检查网页结构
4. WHEN 用户遇到 Cloudflare 阻止，THE 系统 SHALL 建议使用隐秘获取器
5. WHEN 用户点击"查看日志"，THE 系统 SHALL 显示完整的错误日志和堆栈跟踪
6. WHEN 用户点击"重试"，THE 系统 SHALL 使用相同配置重新执行失败的任务

### 需求 11：用户界面响应性与可访问性

**用户故事**：作为一个用户，我想在各种设备上使用 Web UI，并且界面应该易于导航和理解。

#### 验收标准

1. THE Web UI SHALL 在桌面、平板和移动设备上正确显示
2. THE Web UI SHALL 支持浅色和深色主题
3. WHEN 用户使用键盘导航，THE 系统 SHALL 支持 Tab 键、Enter 键和 Escape 键
4. THE Web UI 中的所有交互元素 SHALL 具有清晰的焦点指示器
5. THE Web UI 中的所有图像 SHALL 包含替代文本
6. THE Web UI 中的所有表单 SHALL 包含标签和错误消息

### 需求 12：后端 API 与 WebSocket 通信

**用户故事**：作为一个系统，我需要提供 REST API 和 WebSocket 端点，以便前端应用可以与 Scrapling 框架通信。

#### 验收标准

1. THE 后端 SHALL 提供 REST API 端点用于任务管理（创建、读取、更新、删除）
2. THE 后端 SHALL 提供 WebSocket 端点用于实时进度推送
3. WHEN 前端发送 API 请求，THE 后端 SHALL 返回 JSON 格式的响应
4. WHEN 任务状态变化，THE 后端 SHALL 通过 WebSocket 推送更新
5. THE 后端 SHALL 验证所有 API 请求的身份和权限
6. WHEN 后端发生错误，THE 后端 SHALL 返回适当的 HTTP 状态码和错误消息

### 需求 13：数据持久化与备份

**用户故事**：作为一个用户，我想确保我的任务配置和爬取结果被安全地保存，以便在系统故障时不会丢失数据。

#### 验收标准

1. THE 系统 SHALL 将所有任务配置存储在数据库中
2. THE 系统 SHALL 将所有爬取结果存储在数据库中
3. WHEN 用户导出数据，THE 系统 SHALL 创建数据的备份副本
4. WHEN 系统发生故障，THE 系统 SHALL 在恢复后保留所有数据
5. WHERE 用户启用自动备份，THE 系统 SHALL 每天自动备份数据

### 需求 14：性能与可扩展性

**用户故事**：作为一个系统管理员，我想确保 Web UI 能够处理多个并发用户和大量数据，以便支持生产环境的使用。

#### 验收标准

1. THE Web UI 页面加载时间 SHALL 不超过 2 秒
2. THE API 响应时间 SHALL 不超过 500 毫秒（不包括爬取时间）
3. THE 系统 SHALL 支持至少 100 个并发用户
4. WHEN 用户导出大型数据集（>100MB），THE 系统 SHALL 使用流式传输而不是一次性加载
5. THE 系统 SHALL 使用缓存减少数据库查询

### 需求 15：安全性

**用户故事**：作为一个用户，我想确保我的数据和凭证被安全地保护，以便防止未授权访问。

#### 验收标准

1. THE 系统 SHALL 使用 HTTPS 加密所有通信
2. THE 系统 SHALL 使用 JWT 令牌进行身份验证
3. THE 系统 SHALL 对所有密码进行加密存储
4. THE 系统 SHALL 验证所有用户输入以防止 SQL 注入和 XSS 攻击
5. THE 系统 SHALL 实现速率限制以防止暴力攻击
6. THE 系统 SHALL 记录所有用户操作用于审计

### 需求 16：日志与监控

**用户故事**：作为一个系统管理员，我想监控系统的运行状态和性能，以便及时发现和解决问题。

#### 验收标准

1. THE 系统 SHALL 记录所有 API 请求和响应
2. THE 系统 SHALL 记录所有爬虫任务的执行日志
3. THE 系统 SHALL 记录所有错误和异常
4. WHEN 系统性能下降，THE 系统 SHALL 生成警告日志
5. THE 系统 SHALL 提供日志查询和分析界面

### 需求 17：文档与帮助

**用户故事**：作为一个新用户，我想获得清晰的文档和帮助，以便快速学习如何使用 Web UI。

#### 验收标准

1. THE 系统 SHALL 提供在线帮助文档
2. THE 系统 SHALL 在关键功能旁提供"?"帮助按钮
3. WHEN 用户点击帮助按钮，THE 系统 SHALL 显示相关的帮助内容或教程
4. THE 系统 SHALL 提供常见问题解答（FAQ）页面
5. THE 系统 SHALL 提供视频教程链接

### 需求 18：通知与告警

**用户故事**：作为一个用户，我想在任务完成或发生错误时收到通知，以便及时了解任务状态。

#### 验收标准

1. WHEN 任务完成，THE 系统 SHALL 显示浏览器通知
2. WHERE 用户启用邮件通知，THE 系统 SHALL 发送任务完成邮件
3. WHEN 任务失败，THE 系统 SHALL 立即显示错误通知
4. WHEN 用户点击通知，THE 系统 SHALL 导航到相关任务页面

### 需求 19：集成与扩展

**用户故事**：作为一个开发者，我想通过 API 集成 Web UI 到其他系统，以便构建自定义工作流。

#### 验收标准

1. THE 系统 SHALL 提供完整的 REST API 文档
2. THE 系统 SHALL 支持 API 密钥认证用于第三方集成
3. THE 系统 SHALL 提供 Webhook 支持用于事件通知
4. THE 系统 SHALL 提供 Python SDK 用于程序化访问
