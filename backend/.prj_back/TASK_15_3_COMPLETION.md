## 需求分析

根据设计文档中的属性 9 定义：

> *对于任何* 自定义请求头的添加/编辑/删除操作序列，最终请求头列表应该按顺序反映所有操作，没有重复且值正确。

这要求系统能够：
1. 创建自定义请求头（添加操作）
2. 更新自定义请求头（编辑操作）
3. 删除自定义请求头（删除操作）
4. 确保最终状态正确反映所有操作
5. 防止重复的请求头键
6. 保持操作的一致性

## 实现内容

### 1. 请求头服务 (`app/services/header.py`)

创建了完整的请求头服务，包含以下功能：

#### 核心方法

- **`validate_header_data()`**: 验证请求头数据
  - 验证键不为空且长度 ≤ 255 字符
  - 验证值不为空且长度 ≤ 1000 字符
  - 返回验证结果和错误消息

- **`create_header()`**: 创建新请求头
  - 验证输入数据
  - 创建数据库记录
  - 返回创建的请求头对象

- **`get_header()`**: 按 ID 获取请求头
  - 按用户 ID 和请求头 ID 过滤
  - 抛出 NotFoundError 如果不存在

- **`get_headers()`**: 获取请求头列表（分页）
  - 支持分页查询
  - 返回请求头列表和总数

- **`get_all_headers()`**: 获取所有请求头（无分页）
  - 获取用户的所有请求头

- **`update_header()`**: 更新请求头
  - 验证更新数据
  - 更新数据库记录
  - 返回更新后的请求头

- **`delete_header()`**: 删除请求头
  - 按用户 ID 和请求头 ID 删除
  - 记录删除操作

- **`get_headers_dict()`**: 获取请求头字典
  - 返回键值对映射

- **`delete_all_headers()`**: 删除所有请求头
  - 删除用户的所有请求头
  - 返回删除数量
